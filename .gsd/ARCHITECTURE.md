## Components

### Core Layout (`index.html`)
- **Purpose:** Structure of the single page app, including navigation, canvas elements for simulations, and all UI sections.
- **Location:** `/index.html`
- **Dependencies:** `modern-theme.css`, `style.css`, `main.js`, `background3d.js`, Three.js r128 (CDN)

### Main Logic (`main.js`)
- **Purpose:** Handles all 3D interactives, UI logic, simulations (JWST, Orrery, Starlink, Mission Theater, Reentry Forge, Exoplanet Transits, Black Hole), and the floating AI terminal draggable logic.
- **Location:** `/main.js`
- **Dependencies:** Three.js

### Cinematic Background (`background3d.js`)
- **Purpose:** Decoupled, global High-Fidelity 3D background engine running on a fixed canvas to provide a continuous space vista.
- **Location:** `/background3d.js`
- **Dependencies:** Three.js

### Styles (`style.css` & `modern-theme.css`)
- **Purpose:** Enforces the "Glassmorphism", neon-lit aesthetic and responsive grid layouts.

## Data Flow

1. Initial page load renders `index.html` and static UI components.
2. `background3d.js` locks onto `#bg-canvas` and kicks off the 3D rendering loop.
3. `main.js` builds individual Three.js scenes onto their respective section canvases when the canvases become visible.
4. Input and interactions update global variables driving internal mechanics via requestAnimationFrame.
